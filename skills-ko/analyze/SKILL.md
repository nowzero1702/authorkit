# authorkit.analyze — 레퍼런스/원고 분석

레퍼런스 또는 원고 파일을 분석하여 구조, 그림/도표, 핵심 개념, 이슈를 추출합니다.

## 사용법 예시

- "레퍼런스 분석해줘" → 전체 레퍼런스 분석
- "원고 5장 분석해줘" → 원고의 특정 챕터 분석
- "이 PDF 구조 추출해줘" → 특정 파일 분석

## 수행 작업

### 1. 구조 추출
파일 형식에 따라 목차/헤딩 구조를 추출합니다.
- PDF: 텍스트에서 "Chapter", "장", "절" 패턴 탐색
- docx: Heading 스타일 기반 구조 추출
- hwpx: ZIP 해제 후 XML 파싱
- txt/md: 마크다운 헤딩(#) 기반

### 2. 그림/도표 목록 추출
캡션이 있으면 캡션 기반, 없으면 본문에서 도해 가능한 개념을 탐색합니다.
탐색 패턴: 순서/단계, 비교 구조, 흐름 설명, 구성요소 나열

### 3. 핵심 개념/용어 추출
괄호 표기, 영어 병기, 굵은 글씨, 약어 정의 패턴을 탐지합니다.

### 4. 분량 분석
섹션별 줄 수/단어 수를 산출합니다.

### 5. 이슈 발견
빈 섹션, 번호 불일치, 용어 비일관성, 상호참조 깨짐, 전방참조를 탐지합니다.

## 산출물

분석 결과는 `authorkit/references/ref-XXX/analysis.md` 또는
`authorkit/manuscript/analysis.md`에 저장됩니다.
PDF인 경우 그림 페이지를 개별 PDF로 추출하여 `figures/` 폴더에 저장합니다.

## 지원 형식

| 형식 | 처리 방법 |
|------|----------|
| .pdf | pypdf + pdfplumber |
| .docx | python-docx |
| .txt | 직접 읽기 (UTF-8) |
| .xlsx | openpyxl |
| .hwpx | ZIP 해제 후 XML 파싱 |
| .md | 직접 읽기 (헤딩 파싱) |
