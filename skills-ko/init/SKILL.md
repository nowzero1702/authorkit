# authorkit.init — 프로젝트 초기 설정

이 스킬은 책 집필 프로젝트의 초기 환경을 설정합니다.
콘솔 질문 대신 질문지 md 파일을 생성하여 작가가 IDE에서 편하게 답변할 수 있도록 합니다.

## 실행 흐름

### 첫 실행 — 질문지 생성

`authorkit/setup-questionnaire.md`가 존재하지 않으면 질문지를 생성합니다.

1. `authorkit/` 디렉터리 생성
2. 아래 질문지 템플릿을 `authorkit/setup-questionnaire.md`로 저장
3. 사용자에게 안내: "질문지가 생성되었습니다. IDE에서 열어 답변을 작성한 뒤, 다시 실행해 주세요."

### 재실행 — 답변 파싱 및 세팅

질문지 파일이 존재하고 답변이 작성된 경우:

1. 질문지 파싱 (체크박스 `[x]`, `> 답변:` 텍스트, 경로 유효성)
2. 필수 항목 누락 시 안내 후 중단
3. 프로젝트 구조 생성:
   - `authorkit/constitution.md` — 문체/용어 규칙
   - `authorkit/glossary.md` — 용어집 초안
   - `authorkit/structure.md` — 목차 구조
   - `authorkit/references/` — 레퍼런스 경로 기록
   - `authorkit/manuscript/` — 원고 경로 기록
   - `authorkit/drafts/` — 목차 기반 챕터 폴더
4. 기존 원고가 있으면 문체 자동 분석 → constitution에 반영
5. 완료 안내 + 다음 단계 가이드

## 카테고리별 기본 constitution

기존 원고가 없고 문체 설정이 비어있으면, 카테고리에 따라 기본값을 적용합니다.

### 전공 기술서적
- 종결 어미: ~합니다, ~입니다
- 팁 문체: ~이다 (문어체)
- 용어: 한국어(영어, 약어)
- 참조: "N장 참고", "N-M절에서 설명"

### 교육용 실용서
- 종결 어미: ~합니다, ~해요
- 팁 문체: ~입니다
- 용어: 한국어(영어)
- 참조: "N장에서 자세히"

### 일반 독서용
- 종결 어미: ~이다, ~한다 또는 ~합니다
- 용어: 한국어 위주
- 참조: 자유 형식

## 기존 원고 문체 자동 분석

기존 원고가 등록된 경우 다음을 자동 분석합니다:

1. 종결 어미 빈도 (~합니다 vs ~이다 vs ~해요)
2. 전문 용어 표기 패턴 (괄호, 영어 병기)
3. 보충 요소 패턴 (이지스팁, 더 알아보기 등)
4. 존대 수준 (~봅시다 vs ~볼까요 vs ~해보세요)

분석 결과를 constitution.md에 기본값으로 채우고,
"자동 분석 결과입니다. 수정이 필요하면 constitution.md를 편집하세요."라고 안내합니다.

## Windows 호환성

- 경로: 백슬래시와 슬래시 모두 지원
- 인코딩: UTF-8
- 줄바꿈: LF 또는 CRLF 모두 파싱 가능
