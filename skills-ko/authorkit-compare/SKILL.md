# authorkit.compare — 레퍼런스 ↔ 원고 대조

레퍼런스와 원고를 대조하여 내용 분류, 그림 매핑, 개연성 검증을 수행합니다.

## 선행 조건

`/authorkit.analyze`가 레퍼런스와 원고 모두에 대해 실행되어 있어야 합니다.

## 수행 작업

### 1. 구조 비교 표
레퍼런스의 챕터 구조와 원고의 챕터 구조를 나란히 비교합니다.

### 2. 내용 분류
각 챕터/절 단위로 세 가지로 분류합니다:
- **가져올 것**: 레퍼런스에서 참고하여 원고에 반영할 내용
- **생략할 것**: 대상 독자 수준 초과 등의 이유로 넣지 않을 내용
- **독창 콘텐츠**: 레퍼런스에 없지만 원고에 필요한 내용

### 3. 그림 매핑
레퍼런스의 그림이 원고 어디에 배치될지 매핑합니다.
처리 방식(간소화, 텍스트 블록도 변환, 독창 제작)도 결정합니다.

### 4. 개연성 검증
- 선수지식 순서: A를 설명하기 전에 A가 의존하는 B가 먼저 나오는지
- 전방참조: 아직 설명하지 않은 개념을 참조하지 않는지
- 누락: 핵심 개념이 빠지지 않았는지
- 깊이 일관성: 특정 챕터에서만 지나치게 깊지 않은지

### 5. 절 순서 제안
개연성에 심각한 문제가 있으면 재배치를 제안합니다.

## 멀티 레퍼런스

레퍼런스가 여러 개이면 각각 대조한 뒤 통합 뷰를 제공합니다.
