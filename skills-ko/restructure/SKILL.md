# authorkit.restructure — 구조 재배치

원고의 챕터/절 순서를 분석하고 재배치를 제안합니다.

## 수행 작업

### 1. 현재 구조 분석
각 절의 핵심 개념과 의존하는 선수지식을 파악합니다.

### 2. 의존 관계 그래프
절 간 의존 관계를 방향 그래프로 모델링합니다.

### 3. 개연성 검증
현재 순서에서 의존 관계가 깨지는 곳을 탐지합니다:
- A가 B에 의존하면, B가 A보다 먼저 와야 합니다
- "앞서 배운 X"라고 참조하면, X가 실제로 앞에 있어야 합니다

### 4. 재배치 제안
위상 정렬(Topological Sort)로 유효한 순서를 결정합니다.
동일 수준에서는: 레퍼런스 순서 참고 → 난이도 오름차순 → 분량 균형

### 5. 영향 분석
재배치 시 수정이 필요한 상호참조를 목록화합니다.

### 6. 적용 (apply 모드)
- drafts 폴더의 절 번호 변경
- structure.md 갱신
- 상호참조 수정 목록 생성

## 사용법 예시

- "6장 구조 분석해줘" → 재배치 제안
- "재배치 적용해줘" → 실제 폴더/파일 변경
- "순환 의존 있는지 확인해줘" → 의존 관계 검증만

## 순환 의존 감지

A → B → A 같은 순환이 발견되면:
1. 순환 관계를 작가에게 보고
2. 어떤 의존을 끊을지 선택 요청
3. 선택에 따라 재배치 진행
